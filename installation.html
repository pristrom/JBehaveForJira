Installation

  • Install Jira Plugin
  • Enable Plugin for a Jira project
  • Download Plugin Client
  • Configure JBehave runner

The "JBehave for Jira" plugin can be installed via the Jira's built in "Add-on Management" functionality

1. Navigate to "Add-ons" section from the administration menu and then click on "Find New Add-ons" from the menu on the left


2. In the search field type in 'JBehave for Jira' and then click on either "Buy Now" or "Free Trial" against the "Jbehave for Jira" plugin entry. Follow the prompts from then on to complete the installation.

Note: if you are running an older version of Jira that does not come with the built in plugin manager functionality then please contact us for an alternative way on how to install this plugin in your Jira instance.


Enable Plugin for a Jira project

1. Once the plugin has successfully been installed you will need to enable it for a specific Jira project. To do so go the "Manage Add-ons" page in the Administration area and then click on "Configure" button against "JBehave for Jira" plugin entry.

2. In the opened plugin configuration screen, select General tab and then select the projects that you want to have the plugin enabled for and click the relevant arrow button to move it to the enabled projects box and then click Apply Changes button.


Download Plugin Client

The plugin requires a Java client to work, which comes as in the form of a Jar file that you need to have on your class path when running your JBehave tests. It can be downloaded from this location.

Maven users - the client library will soon be available for download from the maven central repository. Watch this space for an update.


Configure JBehave runner

For the plugin client to exchange data with the plugin running in your Jira instance you need to make the following changes to your JBehave Embedder (runner):

1. Use JiraStoryPathsFinder implementation to find story paths

2. Set JiraStoryLoader as the StoryLoader implemenation to use in your embedder's configuration

4. Set JiraStepdocReporter as the StepDocReporter implemenattion to use in your embedder's configuration, and then call reportStepdocs method on the Embedder instance after executing your story tests

A complete example project that contains the above changes applied can been downloaded from this location.
